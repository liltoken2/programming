В файле in.txt записан электронный документ в формате JSON. При этом в документе не встречаются массивы, только объекты.
Требуется:
1. Определить количество различных ключей в документе, а также максимальную вложенность каждого из них. Вывести собранную информацию в файл out.txt в формате "ключ количество максимальная_вложенность", по одному тегу в строке.
2. Упорядочить строки файла out.txt по названиям тегов по алфавиту в обратном порядке и записать в out1.txt.

Файл и память целиком не считывать, работать построчно.
Использовать только функции работы со строками, без применения библиотек для парсинга JSON, регулярных выражений и т.д.
В рамках данной задачи ключом может быть строка латинских букв, заключённая в кавычки. Значением может быть либо натуральное число, либо строка из латинских букв в кавычках, либо объект.

Пример

in.txt:
{"c":{"bb":
{"a":123, "bb":44}}
}

out.txt:
c 1 1
bb 2 3
a 1 3

out1.txt:
c 1 1
bb 2 3
a 1 3
